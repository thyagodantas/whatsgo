docs: add CALL.md describing meowcaller integration

whatsgo only implements the receive side of WhatsApp calling: it parses
<call> stanzas, dispatches CallOffer/CallAccept/CallTerminate events,
and exposes RejectCall. Full call lifecycle (answer, place outbound,
audio/video) lives in the external meowcaller library.

Document the integration pattern: callers add a go.mod replace that
swaps go.mau.fi/whatsmeow for github.com/thyagodantas/whatsgo, so
meowcaller's *whatsmeow.Client reference resolves to the same struct as
*whatsgo.Client. Without that replace, the two types are distinct at
compile time even though their memory layouts match.

Also call out the existing receive-only support (RejectCall + event
handlers) for callers that don't need full calling.

(Initially tried to add meowcaller as a whatsgo dependency with the
replace inside whatsgo/go.mod, but Go only honors the replace from the
root module that drives the build, so the replace has to live in the
caller's go.mod.)
