sqlstore: revert table name prefix from whatsgo_ to whatsmeow_

When the module was renamed from whatsmeow to whatsgo, a global
text replace turned every occurrence of "whatsmeow" (including the
SQL table name prefix) into "whatsgo" across the whole repo.

That worked fine for Go identifiers, but it broke the SQL schema:
00-latest-schema.sql creates tables named whatsmeow_device,
whatsmeow_identity_keys, etc., while the regenerated Go code in
container.go / store.go / lidmap.go was referencing them as
whatsgo_device etc. Result: Upgrade() created the tables, but every
read/write failed with "no such table: whatsgo_device".

The 00-latest-schema.sql and per-upgrade SQL files were left
untouched because they are loaded by path, not by module name,
so the rename pass didn't reach them.

Fix: in the three .go files that reference the SQL table names,
revert the prefix from whatsgo_ to whatsmeow_ so they match the
schema. This also preserves forward-compat with anyone who already
had a whatsmeow_device / whatsmeow.db lying around.

Found this when the first real QR scan completed but the post-pair
save failed with:
    Failed to pair device: failed to save device store:
    SQL logic error: no such table: whatsgo_device (1)
