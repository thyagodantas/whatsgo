store: add FetchLatestWAVersion helper

whatsgo now exposes a function that hits the WA check-update endpoint
and returns the current client version as a WAVersionContainer. This
lets the caller decide when to fetch (typically at startup) and how to
apply it (via store.SetWAVersion).

Typical usage:

    version, err := store.FetchLatestWAVersion(ctx, http.DefaultClient)
    if err != nil { /* fallback to bundled */ } else {
        store.SetWAVersion(version)
    }

The helper does no caching of its own — caching is the caller's job, so
the function stays a thin pure helper with no global state.

Also: tweak the doc comment on the bundled waVersion fallback in
clientpayload.go to make it clear it's a fallback, and point to the
new helper as the recommended way to stay in sync.
