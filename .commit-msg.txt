send: inject MessageSecret for interactive messages

Upstream only injects a MessageContextInfo.MessageSecret (32 bytes) when
the client is in "bot mode" (Business API / InitialPairBot). Most callers
don't use that, so when they send ButtonsMessage / ListMessage /
InteractiveMessage from a regular session, the recipient can't decrypt
the payload (renders empty / "Aguardando mensagem").

whatsgo now injects the secret automatically in getMessageContent for any
interactive message, regardless of bot mode. Non-interactive messages
(text, media) are untouched, so the existing behavior is preserved.

The helper also follows the ViewOnceMessage / ViewOnceMessageV2 /
EphemeralMessage wrappers, so interactive messages wrapped in those
container types are handled too.
