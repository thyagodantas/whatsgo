proto: regenerate all .pb.go with protoc-gen-go v1.36.11

The committed .pb.go files were last regenerated by an older
protoc-gen-go whose seed layout is incompatible with
google.golang.org/protobuf v1.36.11 (which is the version Go modules
now resolve to for the engine). The first symptom is a runtime
panic during package init:

    panic: runtime error: slice bounds out of range [-5:]
    ... file_xxx_xxx_proto_init ...
    ... protobuf/internal/filedesc.(*File).unmarshalSeed ...

go build ./... still succeeds because each package is compiled in
isolation, but go run / go test trip the panic because every
proto/xxx package's init() runs against the same protobuf runtime.

Fix: regenerate all 57 .proto files with the local
protoc-25.3 + protoc-gen-go v1.36.11. Imports, syntax and the
existing code that consumes these types are unchanged; only the
generated encoding of the file descriptors differs.

Verified with a minimal program that imports every regenerated
package: init() runs clean.
