store: drop FetchLatestWAVersion (wrong endpoint)

The previous FetchLatestWAVersion in store/waversion.go hit
https://web.whatsapp.com/check-update, which returns the version
advertised to the end-user web client (so the browser can prompt
"please refresh"). That number, when set as our client version,
gets rejected by the messaging server with HTTP 405 on connect:

    Client outdated (405) connect failure (client version: 2.2413.51)

The correct version is exposed by whatsgo.GetLatestVersion, which
fetches the WA web HTML and parses the "client_revision":NNNNN
field. That's the revision the messaging server actually accepts.

So:
  * Delete the broken store.FetchLatestWAVersion helper.
  * Document whatsgo.GetLatestVersion as the supported way to
    discover the current version (already existed in update.go,
    just not pointed at).

Backend usage (in manager.go): use whatsgo.GetLatestVersion +
store.SetWAVersion, not the removed helper.
